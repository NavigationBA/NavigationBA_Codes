    Topic 5.5 - 窗口函数中的进阶计算函数¶

    在窗口函数中，可以使用的计算函数很多

    • 我们之前介绍过一些简单的计算函数，包括：

      • COUNT：计算窗口中框出的函数
      • SUM：计算窗口中框出的数据的总和
      • AVG：计算窗口中框出的数据的平均值
      • MIN：计算窗口中框出的数据的最小值
      • MAX：计算窗口中框出的数据的最大值

    • 除了我们之前介绍的一些基本的计算函数之外，还有一些进阶的计算函数

    • 这些函数可以帮助我们在窗口函数中进行进阶计算，而且实用性很高，我们这一节来介绍一下这些进阶的计算函数

    1. 取上下行数据¶

    (1) 取上下行数据的函数的语法¶

    在窗口函数中，我们可以使用一些函数来取上一行或下一行的数据：

    • LAG(column, n) 函数：LAG 函数可以让我们在窗口函数中访问当前行的前 n 行的数据
    • LEAD(column, n) 函数：LEAD 函数可以让我们在窗口函数中访问当前行的后 n 行的数据

    这两个函数的使用语法如下：

    -- 取上一行数据
LAG(column, n)  OVER (PARTITION BY partition_column ORDER BY order_column) AS new_column
-- 取下一行数据
LEAD(column, n) OVER (PARTITION BY partition_column ORDER BY order_column) AS new_column
    • 大家可以看出来，这两个函数并不需要指定 ROWS 语句，也就是说，我们不需要指定窗口范围
    • 这两个函数会自动根据我们指定的 n 来取上一行或下一行的数据

    比方说我们有以下数据，我们按照商品进行分组，之后取每个商品上月销量，作为一个新列：

    注意，我们之前讲过：

    • 有些窗口函数，例如 SUM 函数、AVG 函数等，这些函数在计算时，当计算窗口范围小于指定范围的时候，它们会按照实际的窗口大小进行计算
    • 但是 LAG 函数和 LEAD 函数在计算时，向上或向下取不到数据时，会直接返回 NULL

    WITH
Months AS (
    SELECT DISTINCT DATETRUNC(MONTH, InvoiceDate) AS Month
    FROM Invoice
),
Billing_Country_Group AS (
    SELECT DISTINCT
        CASE 
            WHEN BillingCountry = 'Australia' THEN 'AU'
            WHEN BillingCountry IN ('USA', 'United Kingdom', 'Canada', 'New Zealand') THEN 'Non_AU_English_Speaking'
            ELSE 'Other'
        END AS CountryGroup
    FROM Invoice
),
Billling_Country_Month AS (
    SELECT
    M.Month,
    B.CountryGroup
    FROM Months AS M

            WHEN BillingCountry = 'Australia' THEN 'AU'

            ELSE 'Other'
        END AS CountryGroup
    FROM Invoice
),
Invoice_Country_Group_Summary AS (
    SELECT 
        DATETRUNC(MONTH, InvoiceDate) AS Month,
        CountryGroup,
        SUM(Total) AS Total
    FROM Invoice_Country_Group
    GROUP BY DATETRUNC(MONTH, InvoiceDate), CountryGroup
)
SELECT 
    B.Month,
    B.CountryGroup,
    COALESCE(S.Total, 0) AS Total
FROM      Billling_Country_Month        AS B
LEFT JOIN Invoice_Country_Group_Summary AS S ON B.Month = S.Month AND B.CountryGroup = S.CountryGroup
ORDER BY B.Month, B.CountryGroup
;

    2. 取指定行数据¶

    (1) 取指定行数据的函数的语法¶

    在一个窗口中，如果想取特定行的数据，可以使用以下函数：

    • FIRST_VALUE(column) 函数：在窗口函数中访问当前窗口中的第一行的数据
    • LAST_VALUE(column) 函数：在窗口函数中访问当前窗口中的最后一行的数据
    • 在有些 SQL 数据库中，还有一个 NTH_VALUE(column, n) 函数，可以在窗口函数中访问当前窗口中的第 n 行的数据，但是 Azure SQL 数据库不支持这个函数😅

    这两个函数的使用语法如下：

    -- 取窗口中第一行数据
FIRST_VALUE(column) OVER (PARTITION BY partition_column ORDER BY order_column ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING) AS new_column
-- 取窗口中最后一行数据
LAST_VALUE(column)  OVER (PARTITION BY partition_column ORDER BY order_column ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING) AS new_column

    这个函数语法有些复杂，咱们一点点来解读：

    • 首先，这里为什么强调 ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING 这个窗口范围呢

    • 这是因为，我们通常想要同一个分组中的第一行数据或最后一行数据，而不是当前行的数据

    • 例如商品销量这个例子，我们想要每个商品在一定时间范围内，头一个月的销量数据和最后一个月的销量数据：

    当我们指定了 N-1 行的数据时，那么本质上和向上和向下取数据是类似的：

    • 在 FIRST_VALUE 函数中，指定 N-1 PRECEDING，就相当于带自动填充 NULL 值的 LAG 函数，我们在商品销量例子中看一下这个效果，比方说窗口大小为3：
    • 在 LAST_VALUE 函数中，指定 N-1 FOLLOWING，就相当于带自动填充 NULL 值的 LEAD 函数，我们在商品销量例子中看一下这个效果，比方说窗口大小为3：

    另一方面来说，如果指定了 CURRENT ROW，就相当于取当前行的数据了：

    • 如果在 FIRST_VALUE 函数中，指定窗口的开始是 CURRENT ROW，就相当于取当前行的数据了，我们在商品销量例子中看一下这个效果，比方说窗口大小为3：
    • 如果在 LAST_VALUE 函数中，指定窗口的结束是 CURRENT ROW，就相当于取当前行的数据了，我们在商品销量例子中看一下这个效果，比方说窗口大小为3：

    我们来总结一下 FIRST_VALUE 函数和 LAST_VALUE 函数的使用：

    • 基本上，这两个函数的使用，都是要使用 ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING 这个窗口范围的
    • 如果我们指定了 N-1 PRECEDING 或者 N-1 FOLLOWING，就相当于带自动填充 NULL 值的 LAG 函数或 LEAD 函数了
    • 如果我们指定了 CURRENT ROW，就相当于取当前行的数据了

    3. 取排名¶

    (1) 取排名的函数的语法¶

    在 SQL 的窗口函数中，我们可以使用以下函数来计算每行在当前窗口中的排名：

    • ROW_NUMBER() 函数可以计算每一行在当前窗口中的排序行号，这个排名是非去重排名
    • RANK() 函数可以计算每一行在当前窗口中的排序排名，这个排名是去重排名
    • PERCENT_RANK() 函数可以计算每一行在当前窗口中的百分比排名，这个排名是去重排名

    我们在商品销量的例子中，看一下按商品分类再按照月份排序的行号，这里我们是升序排序：

    这三个函数的使用语法如下，注意这三个函数体里不加任何参数，需要在 OVER 子句中指定分区列和排序列：

    ROW_NUMBER()   OVER (PARTITION BY partition_column ORDER BY order_column ASC 或 DESC) AS new_column
RANK()         OVER (PARTITION BY partition_column ORDER BY order_column ASC 或 DESC) AS new_column
PERCENT_RANK() OVER (PARTITION BY partition_column ORDER BY order_column ASC 或 DESC) AS new_column
    • 大家其实应该可以想象到了，这个函数其实也是不适用于滑动情景的，因为上一个滑动的窗口会被下一个滑动的窗口覆盖掉，所以这个函数在滑动窗口中是没有什么意义的
    • 但是在分组或拓展窗口中，这个函数是非常有用的，我们可以使用这个函数来对每个分组中的数据进行排序和排名等场景

    (2) 取排名的函数的示例¶

    我们在Chinook数据库中，先准备一下国家组的月度销量数据，再在这个基础上对每个国家组的月度销量进行排名：
