    Topic 5.2 - SQL 中的扩展窗口函数¶

    1. 扩展窗口的概念¶

    扩展窗口其实是滑动窗口的一种特殊情况，扩展窗口指的是以当前行，向前或向后无限延伸的数据集：

    • 向前取数据的场景比较常见
    • 向后取数据也是支持的，但在实际业务中，向后取数据的场景比较少见

    2. 向前取数据的扩展窗口¶

    (1) 向前扩展窗口的概念¶

    这里我们先来看向前延伸，来看一个简单的例子：

    • 比方说，我们有以下销量表，在销量表中，我们想求以下从第一月到当前月的销量总和：
    • 当我们求 2024-06 这一行的销量总和时，窗口框出的数据集是从 2024-01 到 2024-06 的数据集，有6个数据，这6个数据求和，就得到了 2024-06 的销量总和：
    • 当我们求 2024-05 这一行的销量总和时，窗口框出的数据集是从 2024-01 到 2024-05 的数据集，有5个数据，这5个数据求和，就得到了 2024-05 的销量总和：
    • 当我们求 2024-04 这一行的销量总和时，窗口框出的数据集是从 2024-01 到 2024-04 的数据集，有4个数据，这4个数据求和，就得到了 2024-04 的销量总和：
    • 当我们求 2024-03 这一行的销量总和时，窗口框出的数据集是从 2024-01 到 2024-03 的数据集，有3个数据，这3个数据求和，就得到了 2024-03 的销量总和：
    • 当我们求 2024-02 这一行的销量总和时，窗口框出的数据集是从 2024-01 到 2024-02 的数据集，有2个数据，这2个数据求和，就得到了 2024-02 的销量总和：
    • 当我们求 2024-01 这一行的销量总和时，窗口框出的数据集是从 2024-01 到 2024-01 的数据集，有1个数据，这1个数据求和，就得到了 2024-01 的销量总和：

    从上面的例子可以看出，扩展窗口其实就是累计求和：

    • 我们以当前行为中心，向前延伸，框出一个数据集，这个数据集包含了从第一行到当前行的所有数据，然后对这个数据集进行求和，就得到了当前行的累计求和
    • 当然，累计求平均、求最大值、求最小值、累计计数等也可以的

    (2) SQL 中的向前扩展窗口函数¶

    在 SQL 中，向前取数据的扩展窗口函数的语法结构为：

    SELECT
    column1,
    column2,
    ...,
    AGGREGATE_FUNCTION(column) OVER (
        ORDER BY column
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS new_column_name
FROM table_name
;

    我们发现，其实和滑动窗口函数的语法结构是很相似的：

    • 唯一的区别就是，滑动窗口函数中，ROWS BETWEEN 之后的数值是一个固定的数值
    • 而扩展窗口函数中，ROWS BETWEEN 之后的数值是 UNBOUNDED PRECEDING，表示向前无限延伸，直到数据表的第一行

    在 Azure SQL 数据库中，向前扩展窗口函数的 SQL 还可以写成以下简化版本：

    SELECT
    column1,
    column2,
    ...,
    AGGREGATE_FUNCTION(column) OVER (ORDER BY column) AS new_column_name
FROM table_name
;
    • 也就是说，当我们只强调向前拓展窗口，进行类似累加累乘等计算时，可以只写一个 ORDER BY，不写 ROWS BETWEEN
    • SQL 引擎会默认将 ROWS BETWEEN 设置为 UNBOUNDED PRECEDING AND CURRENT ROW，也就是向前拓展窗口的语法结构

    3. 向后取数据的扩展窗口¶

    (1) 向后扩展窗口的概念¶

    当然，扩展窗口也可以向后取数据的，我们就在销售表中看一个简单的例子：

    • 如果当前行是 2024-03，那么窗口框出的数据集是从 2024-03 到 2024-06 的数据集，有4个数据，这4个数据求和，就得到了 2024-03 向后取数据的销量总和：
    • 和上一节当中我们强调的一样，向后取数据会涉及到未来数据泄露的问题，因此，在实际业务中，向后取数据的场景比较少见

    (2) SQL 中的向后扩展窗口函数¶

    在 SQL 中，向后取数据的扩展窗口函数的语法结构为：

    SELECT
    column1,
    column2,
    ...,
    AGGREGATE_FUNCTION(column) OVER (
        ORDER BY column
        ROWS BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING
    ) AS new_column_name
FROM table_name;

    • 其实也和滑动窗口函数的语法结构是很相似的：

      • 滑动窗口函数中，ROWS BETWEEN 之后的数值是一个固定的数值
      • 而扩展窗口函数中，ROWS BETWEEN 之后的数值是 UNBOUNDED FOLLOWING，表示向后无限延伸，直到数据表的最后一行

    • 这里我们就不在数据库中展示向后取数据的扩展窗口函数的示例了

    4. 居中取数据的扩展窗口¶

    (1) 居中扩展窗口的概念¶

    当然，如果我们想求居中取数据，也就是向前向后都取数据的扩展窗口函数也是支持的

    • 但是这个计算没有什么意义，因为向前向后都取数据的扩展窗口函数，框出的数据集就是整个表了，所以每一行的计算结果都是整个表的求和结果：

    • 比方说，对 2024-02 这一行求向前向后都取数据的扩展窗口函数，取到的是全部数据集，对 2024-03 这一行求向前向后都取数据的扩展窗口函数，取到的也是全部数据集，运算结果是一样的，所以这个计算没有什么意义：

    (2) SQL 中的居中扩展窗口函数¶

    在 SQL 中，向前向后取数据的扩展窗口函数的语法结构为：

    SELECT
    column1,
    column2,
    ...,
    AGGREGATE_FUNCTION(column) OVER (
        ORDER BY column
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS new_column_name
FROM table_name;
    • 这里就是将 BETWEEN 和 AND 之后的数值都设置为 UNBOUNDED，表示向前向后都无限延伸，直到数据表的第一行和最后一行
    • 同样，这个我们也不在数据库中展示了，因为算出来没有什么实际意义

    5. 扩展窗口与滑动窗口的组合¶

    我们可以发现，扩展窗口与滑动窗口都是用 ROWS BETWEEN ... AND ... 来控制窗口框出的数据集的范围的：

    • 扩展窗口中可以填入：

      • CURRENT ROW 来表示以当前行为中心
      • UNBOUNDED PRECEDING 来表示向前无限延伸
      • UNBOUNDED FOLLOWING 来表示向后无限延伸

    • 滑动窗口中可以填入：

      • CURRENT ROW 来表示以当前行为中心
      • N PRECEDING 来表示向前延伸 N 行
      • N FOLLOWING 来表示向后延伸 N 行

    因此，我们是可以将扩展窗口与滑动窗口进行组合的：

    • 只要满足历史在前，未来在后，当前在中间的原则，我们就可以将扩展窗口与滑动窗口进行组合：

    • 比方说 ROWS BETWEEN UNBOUNDED PRECEDING AND 1 FOLLOWING 就表示以当前行为中心，向前无限延伸，向后延伸一行

    • 例如我们在销量表中，对 2024-03 这一行求 ROWS BETWEEN UNBOUNDED PRECEDING AND 1 FOLLOWING 的窗口函数，框出的数据集是从 2024-01 到 2024-04 的数据集，有4个数据，这4个数据求和，就得到了 2024-03 的销量总和：

    • 同样，这个操作使用频率也很低，我们就不在数据库中操作了，大家可以自行尝试一下

    6. 扩展窗口函数小结¶

    我们简单总结一下扩展窗口函数：

    • 向前拓展窗口函数的语法结构核心为：ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    • 向后拓展窗口函数的语法结构核心为：ROWS BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING
    • 居中拓展窗口函数的语法结构核心为：ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    • 扩展窗口函数与滑动窗口函数是可以进行组合的，只要满足历史在前，未来在后，当前在中间的原则，我们就可以将扩展窗口与滑动窗口进行组合，例如 ROWS BETWEEN UNBOUNDED PRECEDING AND 1 FOLLOWING 就表示以当前行为中心，向前无限延伸，向后延伸一行
